Section 1: Client Onboarding

Lead intake → CEO approval → KYC → active bidder

Presenter summary

  • Prospective client enters through referral, marketing, ops, walk-in, or self-registration
  • AE reviews lead first
  • CEO approves before KYC starts
  • Nomos reviews KYC, then client becomes active bidder

What’s happening

  • Lead comes in
  • AE checks fit + explains onboarding
  • CEO approves/rejects
  • If approved: account created + portal access
  • KYC docs collected
  • Nomos reviews
  • If approved: activate bidder
  • If rejected: block/close case

Expected automation

  • Client record creation
  • Source/status tagging
  • Welcome + password setup
  • KYC status tracking
  • Internal notifications
  • Bidder activation / restriction

Extra automation + questions

  • Structured lead queue
  • CEO approval reminders
  • Missing KYC doc reminders
  • Dashboard for pending approvals / KYC
  • Question: does self-registration follow the exact same path?
  • Question: where are leads stored today?

Section 2: Vendor Onboarding

Vendor lead → CEO approval → KYC → VCA signing → active vendor

Presenter summary

  • Prospective vendor is reviewed by AE
  • CEO approves before KYC
  • Nomos clears compliance
  • Vendor Consignment Agreement is signed via Click & Sign

What’s happening

  • AE qualifies vendor
  • CEO approves/rejects
  • If approved: account + portal access
  • KYC docs collected + reviewed by Nomos
  • VCA initiated
  • Vendor signs, then CEO signs
  • Vendor becomes active consignor

Expected automation

  • Vendor record creation
  • Status tracking
  • KYC workflow progression
  • Click & Sign initiation
  • Agreement completion tracking
  • Vendor activation

Extra automation + questions

  • Unsigned VCA reminders
  • Sync signed VCA to portal + storage
  • Question: is the VCA standardized or variable?
  • Question: what exact docs differ by vendor type?

Section 3: Item Consignment

One item per request → AE review → CEO gate → consultant review → ICF signing → inventory

Presenter summary

  • Vendor submits one item per request
  • AE does first review and viewing
  • CEO approves before external authentication
  • After ICF signing, Telos adds item to inventory

What’s happening

  • Vendor submits item request
  • AE evaluates suitability
  • If suitable: vendor books viewing
  • AE inspects item + discusses Buy/Rent Now
  • CEO approves/rejects item
  • Consultant authenticates + values
  • ICF sent via Click & Sign
  • Vendor signs, CEO signs, Telos inventories

Expected automation

  • Consignment form intake
  • One-item-per-request enforcement
  • Viewing slot notifications
  • CEO approval status
  • Authentication status tracking
  • ICF signing workflow
  • Telos inventory trigger

Extra automation + questions

  • Duplicate request detection
  • Appointment reminders
  • Consultant coordination workflow
  • Item stage dashboard
  • Question: what makes an item suitable?
  • Question: what happens if item is not authentic?

Section 4: Item Release

Vendor asks to withdraw item → CEO decision → fees if applicable → Telos handles release

Presenter summary

  • Vendor requests release/withdrawal
  • AE takes it to CEO
  • If approved, fees may apply
  • Telos invoices, tracks payment, and updates item to Released

What’s happening

  • Vendor submits release request
  • AE notifies CEO
  • CEO approves/rejects
  • If no: CEO negotiates to keep item consigned
  • If yes: check fees
  • Telos sends invoice if needed
  • Telos tracks payment and escalates overdue cases
  • Item status becomes Released

Expected automation

  • CEO approval tracking
  • Invoice sent tracking
  • Payment deadline tracking
  • Overdue escalation trigger
  • Released status update

Extra automation + questions

  • Auto-calculate release fees
  • Negotiation outcome tracking
  • Release dashboard
  • Question: what determines applicable fees?
  • Question: when exactly does physical release happen?

Section 6: Auction Execution

Auction goes live → bids happen → sold item payment + payout → unsold item reset

Presenter summary

  • AE launches auction
  • Clients bid live on items
  • After sale, buyer payment and vendor payout begin
  • Telos owns most financial/tracker updates after sale

What’s happening

  • Auction launches
  • Telos marks auction Live
  • Clients bid
  • If item is sold: buyer + AE + Telos + vendor notified
  • 2-day payment follow-up, 7-day escalation
  • Telos marks paid, deducts fees, pays vendor
  • Auction closes
  • Unsold items reset back to Consigned

Expected automation

  • Sold-item notifications
  • 2-day / 7-day payment tracking
  • Payment received triggers
  • Payout triggers
  • Close + unsold reset workflow

Extra automation + questions

  • Real-time sale dashboard
  • Buyer reminder sequence
  • Payout dashboard
  • Question: who confirms payment receipt operationally?
  • Question: what happens to late-paid or disputed sold items?

Section 8: Item Handover to Buyer

Post-sale collection → direct buyer, proxy, or agent paths → Strongroom release

Presenter summary

  • This is the post-sale collection flow
  • Buyer gets a collection window after payment
  • There are direct, proxy, and agent paths
  • Telos releases item from Strongroom and updates location

What’s happening

  • AE prepares Acknowledgement of Obligation
  • If agent required: buyer signs and agent coordinates
  • If no agent: CEO signs Certificate of Ownership manually
  • Buyer is told item can be collected
  • 5-day then 10-day collection windows apply
  • If proxy collects: collector signs, then Telos signs
  • AE uploads docs
  • Telos updates location to Buyer or Agent

Expected automation

  • Buyer collection notification
  • 5-day / 10-day deadline tracking
  • Agent workflow tracking
  • Handover receipt workflow
  • Collection complete status
  • Strongroom release / location update

Extra automation + questions

  • Collection appointment scheduling
  • Proxy verification workflow
  • Overdue collection escalation queue
  • Question: when exactly is an agent required?
  • Question: what happens after 10 business days if still uncollected?

Section 9: Item Handover to Vendor

Unsold item return flow → vendor collection deadlines → Telos releases from Strongroom

Presenter summary

  • This is the return flow for unsold items
  • Vendor must collect within deadline
  • If delayed, AE and Telos get involved, then CEO
  • Telos handles invoice, strongroom release, and location update

What’s happening

  • Vendor is notified about unsold item
  • 10 business day collection window
  • If overdue: AE + Telos notified
  • 20 business day outer window
  • If still overdue: AE consults CEO
  • Telos sends invoice for applicable fees
  • Vendor signs handover receipt, then Telos signs
  • Telos removes item from Strongroom and sets location to Vendor

Expected automation

  • Unsold item notifications
  • 10-day / 20-day tracking
  • Overdue notices
  • Receipt signing workflow
  • Document upload notification
  • Vendor handover complete status

Extra automation + questions

  • Grouped vendor notifications
  • Collection scheduling
  • Storage fee automation
  • Question: what are applicable fees?
  • Question: what happens if vendor still does not collect after 20 days?

Section 10: Complaint Management

Complaint logged → AE handles first → CEO escalation if unresolved

Presenter summary

  • Client or vendor raises a complaint
  • AE is first line of response
  • If unresolved, CEO steps in
  • System should track complaint state and escalation timing

What’s happening

  • Complaint is raised
  • AE speaks to complainant and gathers details
  • AE emails CEO and relevant parties
  • Standard subject format is used
  • If resolved: AE emails resolution
  • If unresolved: escalate to CEO
  • CEO intervenes and confirms resolution
  • Client/vendor confirms closure

Expected automation

  • Complaint registration status
  • Open / resolved / escalated tracking
  • Standard subject / template support
  • 72-hour escalation timer (implementation enhancement)
  • Resolution date tracking

Extra automation + questions

  • Complaint inbox / dashboard
  • SLA tracking
  • Auto-reminders before escalation
  • Question: what complaint categories matter most?
  • Question: should complainants be able to submit complaints through portal forms?

Section 11: New Auction Tracker Setup

Create a new auction tracker entry and initialize the auction lifecycle

Presenter summary

  • CEO and GCEO set up a new auction tracker
  • Initial status is Tentative
  • Tracker then moves through the auction lifecycle
  • System should keep tracker state aligned with real auction state

What’s happening

  • CEO discusses auction with GCEO
  • Template auction tracker is filled
  • Initial statuses are set to Tentative
  • Status lifecycle is: Tentative → Preparing → Marketing → Live → Closed → Completed
  • On Hold can happen from any state

Expected automation

  • New auction creation trigger
  • Initial status setup
  • AE + Telos notifications
  • Status change tracking and notifications
  • Tracker synchronization

Extra automation + questions

  • One-click auction creation from template
  • Status history/audit log
  • Readiness checklist before moving stages
  • Question: which parts of tracker remain manual?
  • Question: should auction creation originate in Shopify or in the tracker?
