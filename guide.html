5. Section 1: Client Onboarding V2

🔄 View Flowchart: Client Onboarding
🔄 V2 Changes: CEO approves lead BEFORE KYC. KYC by Nomos only. Telos CC'd on KYC approval. Shopify creates accounts.

Process Flow

  1. AE receives lead → checks guidelines → introduces Aurelius
  2. Client interested → AE emails CEO requesting approval
  3. CEO approves/rejects (pre-KYC in V2)
  4. If approved → Shopify creates Client Account → welcome email + password
  5. AE uploads KYC documents to DropBox
  6. Shopify notifies AE and Nomos
  7. Nomos conducts vetting (may request additional documents)
  8. KYC approved → Nomos emails AE (CC: CEO, Telos)
  9. AE updates client details → Active Client

Self-registration path: Client signs up via AA website → CEO periodically reviews and removes unwelcome sign-ups (tickets must be refunded).

6. Section 2: Vendor Onboarding V2

🔄 View Flowchart: Vendor Onboarding
🔄 V2 Changes: CEO approves lead BEFORE KYC. Click & Sign for VCA signing. Nomos R&W verification removed. Telos CC'd on approval.

Process Flow

  1. AE receives lead → checks guidelines → introduces Aurelius
  2. Vendor interested → AE emails CEO requesting approval
  3. CEO approves/rejects (pre-KYC in V2)
  4. If approved → Shopify creates Vendor Account → welcome email + password
  5. AE uploads KYC documents to DropBox
  6. Nomos conducts vetting → approves (CC: CEO, Telos)
  7. Click & Sign configures VCA → Vendor signs → CEO signs
  8. Click & Sign notifies all parties → AE uploads signed VCA
  9. Shopify notifies Vendor: onboarding complete + BTP introduction

7. Section 3: Item Consignment V2

🔄 View Flowchart: Item Consignment
🔄 V2 Changes: AE is primary (was CAC). CEO approval gate before evaluation. Click & Sign for ICF signing. Telos manages inventory. One item per request. Buy/Rent Now Price. External Consultant unified role.

Process Flow

  1. Vendor submits Item Consignment Request (one item per request)
  2. AE performs preliminary evaluation
  3. If suitable → AE schedules viewing → Vendor books appointment
  4. AE views item, discusses Buy/Rent Now Price
  5. CEO approves item (NEW gate in V2)
  6. External Consultant evaluates → starting price + authenticity report
  7. If authentic → Click & Sign: Vendor signs ICF → CEO signs ICF
  8. Click & Sign notifies CEO, AE, Telos, Vendor
  9. Telos adds item to Inventory → status = "Consigned"

8. Section 4: Item Release V2

🔄 View Flowchart: Item Release
🔄 V2 Changes: Telos invoices vendor (was CAC). Telos tracks payment + escalates to Nomos. CEO negotiation loop added.

Process Flow

  1. Vendor submits release request → AE emails CEO
  2. CEO approves/rejects → if No, CEO negotiates with Vendor
  3. If fees apply → Telos invoices Vendor → 7-day payment window
  4. If not paid → Telos escalates to Nomos
  5. If paid → Telos confirms → status = "Released"
