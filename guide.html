1. Executive Summary

This document provides complete implementation instructions for the Aurelius Auctions Shopify Platform based on the CEO-approved SOP version 1 (dated 12.02.2026). The system enables luxury auction management through a unified Shopify-powered platform that handles client and vendor onboarding, item consignment, comprehensive auction execution, and post-auction settlement across 11 distinct workflow sections.

The V1 implementation leverages a single Shopify store with clients and vendors differentiated through tags and metafields, not separate stores. The system includes automated KYC processing with Idenfo API for identity screening, separate KYC forms for comprehensive data collection, UAE PASS digital signing integration, and a complete auction management system with eBay-style bidding functionality.

2. Project Overview

🔄 View Flowchart: System Overview

2.1 What We Are Building

A comprehensive auction management platform that enables:

  • Client and vendor onboarding with dual KYC verification (form collection + Idenfo screening)
  • Item consignment workflow with expert authentication and valuation
  • Complete auction preparation including catalog creation and marketing coordination
  • Live auction execution with real-time bidding and Buy Now options
  • Multi-phase item handover processes (to Aurelius, to buyers, to vendors)
  • Comprehensive complaint management system
  • Auction tracker setup and management workflow
  • Automated payment processing and settlement
  • Digital document signing via UAE PASS integration

2.2 Business Model

Auction Structure: Multi-day online auctions with starting prices set by expert evaluation, Buy Now prices at approximately 2x starting price, and incremental bidding over auction duration.

Revenue Streams:

  • Seller's premium: Percentage commission from successful sales
  • Buyer's premium: Additional fee charged to winning bidders
  • Authentication services: Expert verification and certification
  • Consignment fees: Item handling and storage charges

Contract Parameters:

  • Auction duration: Typically 5–7 days per auction
  • Starting bids: Expert-determined based on authentication and valuation
  • Buy Now pricing: Approximately 2x starting bid price
  • Authentication: Mandatory for all luxury items
  • Settlement: T+3 business days post-auction
  • Item handover windows: 15 days before to 1 day before auction start
  • Payment deadlines: 2 business days, escalating to 7 business days
  • Collection deadlines: 7 working days with escalation procedures

5. Section 1: Client Onboarding Flow

🔄 View Flowchart: Client Onboarding Flow

5.1 Process Overview

Clients are prospects who wish to participate in auctions as bidders and buyers. The onboarding process includes dual KYC verification (comprehensive form collection AND Idenfo screening), document collection, and account approval with CEO periodic review capability.

  1. Client signs up via AA website (self-registration) OR lead received by AE
  2. If self-registration, CEO periodically reviews client list and removes unwelcome sign-ups
  3. AE verifies lead details and explains Aurelius process
  4. Initial qualification assessment against guidelines
  5. If qualified, AE creates Client Account in Shopify with basic details
  6. Client receives welcome email with password setup link
  7. Client completes KYC data collection form (personal info, financial standings, occupation, sources of fund/wealth)
  8. Client uploads identity documents for Idenfo screening
  9. Nomos receives both KYC form data and processes Idenfo identity screening
  10. Idenfo performs sanctions, PEP, and document verification screening
  11. If approved, CEO receives approval request from Nomos
  12. Upon CEO approval, AE updates client account and enables auction participation
  13. Client receives onboarding completion notification
  14. Any rejected clients must have tickets refunded

5.2 Shopify Configuration for Clients

  • Customer Type: "Client" (stored in customer tags)
  • KYC Form Status: Separate tracking for form completion
  • Idenfo Screening Status: Separate tracking for identity verification
  • CEO Review Status: Periodic review flag for self-registered clients
  • Approval Level: CEO approval flag
  • Document Status: Form Builder integration for uploads
  • Bidding Permissions: Enabled after full approval

5.3 Dual KYC System Implementation

KYC Data Collection Form (Powerful Form Builder): Personal information, financial standings, occupation details, sources of funds/wealth, risk assessment questionnaire.

Idenfo Identity Screening (Direct API): Document types (Passport, Emirates ID, Trade License), identity verification, sanctions screening, PEP screening, response handling (Pass/Fail/Review).

Critical Distinction: The KYC form collects comprehensive customer data for business assessment, while Idenfo performs technical identity document verification and compliance screening. Both are required for complete onboarding.

6. Section 2: Vendor Onboarding Flow

🔄 View Flowchart: Vendor Onboarding Flow

6.1 Process Overview

Vendors are individuals or entities who consign items for auction. The process includes enhanced KYC with dual verification system, document verification, and Vendor Consignment Agreement execution via UAE PASS. The completion notification includes detailed explanation of Item Consignment process and BTP introduction.

  1. Lead received by AE (marketing, referral, walk-in)
  2. AE verifies details and explains vendor process
  3. Initial qualification against vendor guidelines
  4. If qualified, AE creates Vendor Account in Shopify
  5. Vendor receives welcome email with password setup
  6. Vendor completes comprehensive KYC data collection form
  7. Vendor uploads identity documents for Idenfo screening
  8. Nomos reviews KYC form data and processes via Idenfo screening
  9. Upon KYC approval, CEO receives vendor approval request
  10. CEO approves vendor for onboarding
  11. AE uploads Vendor Consignment Agreement to vendor portal
  12. Vendor signs agreement using UAE PASS digital signature
  13. Nomos verifies signed document (checks Representation & Warranties unchanged)
  14. CEO signs agreement via UAE PASS
  15. Final signed agreement uploaded to vendor portal
  16. Vendor receives onboarding completion notification with BTP introduction

7. Section 3: Item Consignment Flow

🔄 View Flowchart: Item Consignment Flow
MAJOR CHANGE: Primary responsibility has shifted from GDMM to CAC. Terminology updated from "Item Consignment Request" to "Item Consignment Form".

7.1 Process Overview

  1. Vendor uploads Item Consignment Form via portal (one item per form)
  2. CAC receives notification and performs preliminary evaluation (primary responsibility)
  3. If suitable, CAC schedules viewing appointment with vendor
  4. CAC views item and discusses buy-now price with vendor

  8. Nomos verifies signed form (checks R&W unchanged)
  9. CEO signs form via UAE PASS
  10. Item added to inventory with "Consigned" status
  11. CAC coordinates physical item collection and updates inventory

8. Section 4: Item Release Flow

🔄 View Flowchart: Item Release Flow
KEY CHANGES: CEO approval gate required for all releases. Invoice management for applicable fees. 7-day payment tracking with Nomos escalation. No UAE PASS signing for release.

8.1 Process Overview

  1. Vendor submits Item Release Request in Vendor Portal
  2. AE receives notification and negotiates with vendor
  3. If negotiations fail, AE emails CEO requesting approval
  4. CEO approves/rejects release

  7. 7-day payment window; if unpaid → escalate to Nomos
